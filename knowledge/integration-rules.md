# Integration Rules

Интеграции пишутся как отдельные адаптеры.

Правила:

- адаптер не зависит от интерфейса админки;
- секреты не должны попадать в публичный API без маскирования;
- внешние ошибки возвращаются понятным сообщением;
- импортированные данные нормализуются в сущности CMS;
- каждое важное действие пишется в `events`.

Telegram:

- импорт создает черновики;
- если включен `autoPublish`, импорт сразу публикует материал;
- фото сохраняются в `public/uploads/telegram`;
- в Telegram-альбомах первое фото становится основным, остальные показываются внутри поста;
- публикация происходит после проверки в админке;
- webhook лучше polling для продакшена.

Bitrix:

- первая операция: создание лида;
- позже добавить контакты, сделки, статусы и двустороннюю синхронизацию.
